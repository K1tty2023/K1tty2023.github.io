如果我们将来需要添加其他的内容，或者将现有的内容移动到其他文件中，那么使用移动组件而非移动代码的方法会更加合理。这会使组件更加模块化，并且可以在其他文件中复用。

组件导出方式

  1. 默认导出
  2. 具名导出
  3. 二者都有
语法 导出语句 导入语句
默认导出 export default function 组件名(){} import 组件名 from ‘组件路径’
具名导出 export function 组件名(){} import { 组件名 } from ‘组件路径’

当使用默认导入时，我们可以任意自定义组件名称，并且可以获得与默认导出一致的内容。相反，对于具名导入，导入和导出的名字必须一致。

从同一个文件中导出或导入多个组件

如果我们只是想用一个组件中的一小个部分，例如在Gallery中的Profile，我们也可以导出Profile组件。但是由于一个文件中不能定义两个默认导出，因此我们可以选择将其在新文件中默认导出，或者使用具名导出。

在同一个文件中，只能有一个默认导出，但是可以有多个具名导出。

在哪里可以用大括号

  1. 用作JSX标签内的文本：<h1>{name}'s To Do List</h1> 是有效的，但是 <{tag}>Gregorio Y. Zara's To Do List</{tag}> 无效。
  2. 用作紧跟在 = 符号后的 属性：src={avatar} 会读取 avatar 变量，但是 src="{avatar}" 只会传一个字符串 {avatar}。
