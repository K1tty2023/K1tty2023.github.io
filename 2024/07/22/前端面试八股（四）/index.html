什么是虚拟内存

概念

虚拟内存是一种计算机系统内存管理技术，它将硬盘空间作为辅助内存在扩展主存储器的容量，使得运行的程序能够访问比物理内存更大的地址空间。

实现

虚拟内存可以通过页表和页面置换算法来完成。页表用于将逻辑地址映射到物理地址，页面置换算法用于在物理内存不足时将不常用的页面置换置换出去。

优点

提高了内存利用率、增加了程序的地址空间、简化了内存管理。

缺点

带来一定的性能开销。

WebSocket 的原理与实现

概念

WebSocket是一种在单个TCP连接上进行的全双工通信协议，它允许客户端和服务器之间进行实时数据传输。

原理与实现

通过HTTP协议发起握手通信，在握手成功后升级协议到WebSocket，之后客户端和服务器之间可以直接通过该连接进行实时通信。

优点

相比于传统的HTTP请求-响应模式，WebSocket可以实现更低的延迟和更高的效率

WebSocket 与 HTTP 的区别

区别

WebSocket 是一种全双工通信协议，可以实现服务器主动向客户端推送数据。

HTTP是一种半双工通信协议，需要客户端发起请求才能获取数据。

实现

WebSocket在握手成功后会建立持久连接

HTTP是无状态的，每次请求都要重新建立连接

WebSocket的优缺点

优点

  • 实时性好
  • 可以实现服务器主动向客户端推送数据
  • 减少了网络延迟，节省带宽

缺点

  • 在传输数据时会增加额外开销，占用更多资源

HTTP的优缺点

优点

  • 稳定
  • 可靠
  • 广泛

缺点

  • 实时性差

不借助服务端，用WebSocket实现电脑浏览器和手机端的链接

在手机端和电脑端分别建立WebSocket，将其建立的连接指定一个共享的地址，服务器用来转发消息。在手机端和电脑端分别设置消息的处理程序，以便接收和处理从另一端发来的消息。

大文件上传的实现

利用原生API可以通过FormData对象结合XMLHttpRequest来实现大文件上传。

首先将大文件分割成多个小文件（分片），然后使用FormData对象将每个分片添加到表单中，最后通过XMLHttpRequest对象发送这些分片数据到服务器。服务器端接收到分片后在进行合并，最终完成文件的上传。

什么是流式渲染

流式渲染是一种渲染技术，可以在数据还没有完全加载完成时就开始渲染界面，逐步显示内容，提高用户体验。

this 是什么，它的作用 及 应用场景

this 是在面向对象编程中用来代指当前对象的关键字。

在类的方法中，this表示当前的实例对象。

在函数中，this表示全局对象。

在浏览器中，this为window对象。

通过this关键字，我们可以访问当前对象的属性和方法，实现对对象的操作。

new 一个对象或者函数的时候，发生了什么

在进行对象或者函数的实例化时，会经历以下几个步骤：

  1. 分配内存空间：首先在内存中为对象或者函数分配一块连续的内存空间，以便存储其属性和方法
  2. 初始化对象：接着会根据对象或者函数的定义，对其进行初始化，包括设置默认属性值和方法
  3. 将内存地址赋给变量：最后将分配的内存地址赋给对应的变量，以便在程序中引用该对象或函数

当输入了URL，按下回车，浏览器会发生了什么事情

  1. 浏览器会解析URL，将其分解成协议、主机、端口、路径等部分
  2. 浏览器会检查缓存，查看是否有相应的资源缓存
  3. 浏览器建立与服务器的连接，通过DNS解析获取服务器的IP地址，然后通过TCP协议建立连接
  4. 建立连接后，浏览器会发送HTTP请求，请求服务器返回相应的资源
  5. 服务器收到请求后，会处理请求并返回相应的资源
  6. 浏览器接收到响应后，会解析并渲染页面，展示给用户。

如何保证、判断用户的登录态

通过session 或 token 来保证用户的登录态。

session

用户登陆成功后，生成一个session 并将 session id存储在服务器端，同时将session id返回给客户端保持存在cookie中。用户每次请求时，携带session id，服务端验证session id是否有效来判断用户的登录态。

token

用户登陆成功后生成一个token并返回给客户端，客户端在每次请求时在header中携带token，服务端验证token的有效性来判断用户的登录态。

CSS盒模型

盒模型包括什么

  • 外边距(margin) - 清除边框外的区域，外边距是透明的
  • 边框(border) - 围绕在内边距合内容外的边框
  • 内边距(padding) - 清除内容周围的区域，内边距是透明的
  • 实际内容(content) - 盒子的内容，显示文本和图像

标准盒模型（W3C的标准盒模型）

在标准盒模型中，width指content部分的宽度

怪异盒模型（IE盒模型）

在怪异盒模型中，width指content+padding+border的宽度

box-sizing的使用

我们默认使用的正是W3C标准盒模型，而盒模型的选取更倾向于项目合开发者的习惯。如果想要切换盒模型，需要借助box-sizing属性。

WebPack

是什么

WebPack 是一个模块打包工具，可以使用WebPack管理模块，分析模块间的依赖关系，最终编译输出模块为HTML，JavaScript，CSS以及各种静态文件，让开发过程更加高效。

对于不同类型的资源，WebPack有预定的模块加载器loader：

css

解析css的css-loader，style-loader

解析less的less-loader，sass的sass-loader

js

解析将TypeScript转换为JavaScript的ts-loader

解析ES6为ES5的babel-loader

解析JavaScript代码规范的eslint-loader

Vue

解析.vue文件的vue-loader

静态资源

解析图片及音视频的url-loader

解析文件的file-loader

解析json文件的json-loader

基本功能

  1. 代码转换：TypeScript 编译成 JavaScript、ES6转ES5、SCSS 编译成 CSS 等等
  2. 代码语法检测：自动检测代码是否符合语法
  3. 代码分割：打包代码时，可以将代码 切割成不同的块，实现按需加载，降低了初始化事件，提升了首屏渲染效率
  4. 检测代码更新，自动编译，刷新页面
  5. 自动发布
  6. 文件压缩（打包后的js，css，html文件会去掉代码之间的空隙，紧凑显示）
  7. 模块合并：由于模块化开发，一个页面可能会由多个模块组成，所以编译时需要把各个模块合并成一个文件

对于模块化的认识

前端模块化是一种将代码分解为可重复使用的独立片段或模块的策略。这样做保证了代码的整洁、易于维护、减少全局作用域的污染和提高代码复用性。
